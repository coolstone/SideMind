# SideMind AI

一个可直接加载到 Chrome / Edge 的 Manifest V3 浏览器扩展 MVP。产品定位类似 Sider 的核心 Chat 能力：AI 不在独立网页里等你复制粘贴，而是理解当前页面、选中文本和可见截图，在浏览器侧边栏持续协作。

SideMind 源于项目发起人作为 Sider 早期用户的真实经历：订阅价格从早期最高约 999 元，逐步变化到 1399 元成为第二档、最低档达到 1699 元，长期支持的成本越来越难以承受。真正需要保留的是浏览器侧边栏中的 AI 工作方式，因此 SideMind 选择独立实现一个个人可控、模型可选、本地优先且方便扩展的工具。

完整的压缩包安装、DeepSeek API Key 配置、功能说明和二次开发方法，请阅读 [SideMind 使用与扩展指南](SideMind-使用与扩展指南.md)。

## 已实现

- 浏览器原生侧边栏，点击扩展图标即可打开
- 原创 SideMind 品牌标识：思维路径 S 与浏览器 AI 侧边栏节点，并提供 SVG、横向品牌标和 Chrome 多尺寸 PNG
- 自动读取当前网页标题、摘要、标题结构与正文
- 支持把浏览器打开的本地 Markdown、TXT、CSV、JSON、HTML 等文本文件识别为当前页面（需在扩展详情开启“允许访问文件网址”）
- 网页划词浮动工具条：提问、解释、翻译、改写
- 浏览器右键选区可直接总结、解释、翻译、改写，`Alt + Shift + S` 快捷总结整页
- 总结、解释、翻译、改写、结构化提取、自由问答
- 当前标签页截图 + 多模态视觉理解
- 模型回答复制与“写入网页输入框”
- 回答下方操作栏：复制、保存为本地 Markdown、引用追问、重新生成、系统分享、语音朗读、写入网页、确认后删除当前回答
- 代码块独立复制按钮，并显示代码语言
- 本地工件：自动识别 HTML、SVG、Markdown、JSON、CSS、JavaScript/TypeScript、XML、Vue 等代码块，在回答中生成工件卡片
- 独立工件查看器：代码 / 预览 / 分栏三种模式，可本地编辑、自动保存、复制和下载；HTML/SVG/Markdown 使用隔离沙箱预览
- 本地会话历史按聊天空间保存，支持修改聊天标题、空间内全文搜索，以及 Markdown 历史的导入、去重合并和导出，并按 8MB 本地预算裁剪旧记录
- 设置页支持导出和恢复完整本地 JSON 备份，覆盖模型配置、提示词、空间、历史、偏好和本地工件；API Key 默认排除，也可经明确确认后包含
- 顶部爱心入口提供自愿捐赠弹窗，可在微信支付和支付宝二维码之间切换；二维码图片完全保存在扩展本地
- 启动行为可配置为恢复上次打开的会话或始终新建；恢复后同步所属空间，并在历史列表中高亮定位当前会话
- 提示词管理：创建、编辑、分类、搜索、显示/隐藏、拖动或按钮排序，并同步到侧栏快捷入口
- 内置 Sider 同类提示词库：总结、翻译、词汇学习、名师指导、公众号写作、行业分析等 28 个模板；升级时按名称合并，不覆盖个人提示词
- 提示词模板变量：`${lang}` 自动读取设置中的响应语言；`${input}` 优先读取网页选区或输入框；其他 `${变量}` 会自动生成输入项
- 输入框上方显示 5 个常用提示词胶囊，“…”展开完整提示词列表；网页选区会显示为可移除的来源卡片
- 网页选区变更会实时同步到已打开的侧栏；顶部 `+` 可显式创建新聊天，未发送草稿或附件会先确认，生成中禁止误切换
- 服务商优先的多模型配置：先配置服务商的接口、Base URL 和 API Key，再管理其下模型；内置 DeepSeek V4 Flash 与 V4 Pro，侧栏按服务商分组切换
- 多模型答案比较：在任意模型回答下选择另一个已配置模型，复用原问题、上下文、响应语言、思考等级与图片获取替代答案；使用左右箭头在同一位置切换 `1 / N` 个答案，悬停比较图标显示“并排比较”，也可打开独立宽屏页面逐列查看和复制
- 支持 Ollama 本地模型：Base URL 可直接填写 `http://主机:11434`，无需 API Key，并自动使用 OpenAI 兼容的 `/v1/chat/completions`
- API 回答生成期间，发送按钮会变为停止按钮；点击后真正取消后台网络请求并保留已经发送的问题
- 图片、截图和文本附件上传（单文件 3MB、单次最多 6 个、总量 5MB）；支持在输入框按 `⌘V / Ctrl+V` 直接粘贴剪贴板图片，也可不输入文字直接发送附件
- 图片发送后会显示在用户聊天消息中，并以受控尺寸保存在本地聊天历史；恢复会话后仍可查看，点击缩略图可展开
- “阅读此页”主动刷新并挂载网页上下文
- 思考模式：关闭 / 高 / 最大三档，映射到支持的模型 API 参数
- API Key 模式：OpenAI Responses API，以及 DeepSeek、OpenRouter、自定义 OpenAI 兼容接口
- ChatGPT 网页交接模式：组合提示词、写入正常网页、人工确认发送、同步可见回答
- ChatGPT 页面同步按钮只注入 `https://chatgpt.com/`；点击时先在用户手势链中打开侧栏，再异步保存和导入回答
- 跟随系统的明暗主题、键盘操作和基础无障碍支持
- 侧栏提供紧凑、舒适、大、特大四档字号；默认“舒适”比旧版正文与控件文字更大
- 输入框底部控件支持鼠标悬停和键盘聚焦说明，自动显示功能名称、简要用途并避让侧栏边缘

## 安装

1. 打开 Chrome 的 `chrome://extensions/`（Edge 为 `edge://extensions/`）。
2. 打开右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本 `Sider` 目录。
5. 点击 SideMind 图标，浏览器右侧会打开侧边栏。
6. 点击侧边栏右上角设置，选择 API Key 或 ChatGPT 网页交接模式。

如需读取 `file:///...` 本地文件：在 `chrome://extensions/` 点击 SideMind 的“详情”，开启“允许访问文件网址”。重新加载扩展或首次增加该权限后，请刷新已打开的本地文件页。

本项目没有构建步骤，也不依赖 npm；修改文件后，在扩展管理页点击“重新加载”即可。

首次安装会自动打开设置页并提示建立本地备份。正常覆盖同一安装目录再重新加载不会清空 `chrome.storage.local`；如果换目录或重新安装导致扩展 ID 变化，可在设置页的“本地数据与备份”中恢复 JSON 备份。

## 侧栏工作流

- 顶部空间名称用于切换不同项目的聊天空间。
- 输入框底部工具栏的 `+` 用于创建新聊天；点击 SideMind 标志也可执行相同操作。存在未发送文字或附件时会先询问确认。
- 输入框底部工具栏依次提供新聊天、聊天历史、思考模式和阅读此页；连接模式使用紧凑的 `API / 网页` 选择器。
- API 模式开始生成后，右下角发送箭头会变成停止方块；点击即可强制中止当前模型请求。
- 输入框中的模型名称可打开模型列表，并按服务商分组显示。设置页先选择服务商，再只显示该服务商下的模型；服务商的 API 类型、Base URL 和 API Key 由其下模型共同使用。
- “设置 → 侧栏字体大小”提供紧凑、舒适、大、特大四档；保存后已打开的侧栏会立即更新。
- 回形针上传图片或文本文件；输入框支持用 `⌘V / Ctrl+V` 直接粘贴已复制的截图或图片并显示缩略图；截图按钮附加当前可见页面；书本按钮重新读取并挂载当前网页。截图使用持续页面访问权限，更新扩展后需要在扩展管理页重新加载一次。附件采用单文件 3MB、单次最多 6 个、总量 5MB 的限制，避免超过浏览器扩展消息与会话存储预算。
- 原图用于本次模型请求；聊天消息保存压缩后的本地预览图。图片随会话和完整 JSON 备份保存，不上传 SideMind 服务器；较旧图片超过会话预算时保留名称占位。
- “思考”按钮依次循环关闭、高、最大。服务商不支持相应参数时可能返回参数错误，此时关闭思考模式即可。
- 顶部 `✦` 打开提示词列表；点击“管理提示词”进入独立管理页。
- 历史按钮只显示当前空间的对话，并支持全文搜索；每条记录右侧的铅笔按钮可以修改标题，自定义标题在继续聊天后仍会保留。“导入记录”可把 SideMind 导出的 Markdown 去重合并到当前空间；“导出当前空间”会在本地生成一份包含全部问答、时间和关联网页的 Markdown 文件，不包含 API Key。
- 回答下方的下载按钮会直接生成本地 Markdown 文件；该操作不上传回答，也不需要额外服务器。
- 回答下方的“从其他模型获取更智能的答案”会列出除原模型外的可用配置。每次获取都是一次独立 API 请求，会分别消耗对应服务商的 Token 或本地算力；得到替代答案后可用左右箭头切换，或将鼠标移到四角比较图标上并点击打开宽屏比较页。

## 两种连接模式

### 1. API Key 模式

默认使用 OpenAI `POST /v1/responses`，模型预填为 `gpt-5.6-terra`，定位是智能与成本的平衡档。模型可在设置中自由修改。OpenAI 官方建议新项目优先使用 Responses API；Chat Completions 仍受支持，因此本项目也保留了兼容模式。

OpenAI 预设的最大输出 Token 可配置到 `128000`；DeepSeek V4 预设可配置到 `384000`；OpenRouter 和自定义服务不在表单层硬编码上限。该值只是生成预算，并不保证每次都会输出相同长度；日常网页问答和总结建议设置为 `4000`—`16000`，以平衡等待时间与费用。

设置页包含以下预设：

- OpenAI：Responses API，`https://api.openai.com/v1`
- DeepSeek：Chat Completions，`https://api.deepseek.com`，内置 `deepseek-v4-flash` 与 `deepseek-v4-pro`
- OpenRouter：Chat Completions，`https://openrouter.ai/api/v1`
- Ollama：Chat Completions，`http://127.0.0.1:11434`，API Key 可留空；SideMind 自动补全 `/v1/chat/completions`
- 自定义兼容服务：手动填写 Base URL、API 类型和模型 ID

Ollama 默认会校验浏览器来源。如果测试连接返回 HTTP 403，请在运行 Ollama 的电脑上把 SideMind 扩展来源加入 `OLLAMA_ORIGINS` 并重启 Ollama。开发和个人局域网环境可使用 `OLLAMA_ORIGINS=chrome-extension://*`；更严格的做法是在 `chrome://extensions/` 查看 SideMind 的扩展 ID，只放行对应的 `chrome-extension://扩展ID`。如果 Ollama 还需要向局域网开放监听，再按其启动方式配置 `OLLAMA_HOST=0.0.0.0:11434`。不要把未受保护的 Ollama 端口暴露到公网。

图片会以模型 API 规定的多模态数据格式发送，而不是传统表单上传。当前 DeepSeek V4 官方 API 仅支持文本输入；选择 DeepSeek 时，SideMind 会保留已附加图片并提示切换到支持视觉的模型或 ChatGPT 网页交接模式。OpenRouter 与自定义兼容服务是否支持图片，取决于实际路由模型和接口实现。

服务商的模型名称和额度可能变化，请以各自控制台为准。ChatGPT 订阅与 OpenAI API 账单相互独立；订阅 ChatGPT 不会自动提供 API 额度。

### 2. ChatGPT 网页交接模式

此模式使用你已经登录的正常 `chatgpt.com` 页面：

1. SideMind 读取原网页、选区和可选截图，组合完整提示词。
2. 打开或切换到 ChatGPT，将提示词写入输入框。
3. SideMind **不会点击发送**，由你检查并手动确认。
4. 回答完成后，点击 ChatGPT 页面右下角“同步最新回答到 SideMind”，或在侧边栏点击同步按钮。

扩展只读取 ChatGPT 页面当前可见的最新助手消息 DOM。它不读取 Cookie、不调用私有接口、不拦截网络响应，也不绕过登录、订阅或配额。ChatGPT 页面改版后，DOM 选择器可能需要适配。

## 当前边界

- 普通网页及浏览器直接打开的本地文本文件体验最佳；本地文件必须开启“允许访问文件网址”。Chrome 内置页面、扩展商店等受浏览器安全策略保护的页面仍无法读取或写入。
- 当前正文提取是本地启发式规则，不等同于完整阅读器算法。
- 内置 PDF 查看器和 YouTube 字幕尚未做专用适配；可以使用截图视觉理解作为补充。
- 模型回答暂为非流式显示，但等待期间可用停止按钮立即取消后台请求。
- “写入网页”依赖页面的输入框实现，复杂富文本编辑器可能需要站点适配器。
- 当前附件支持 PNG/JPEG/WebP 与 TXT/Markdown/CSV/JSON/HTML；PDF、Office 文档尚未解析。图片历史保存的是本地预览副本，不是未经压缩的原文件。
- ChatGPT 网页模式依赖公开页面 DOM；页面结构变化时，填入或同步功能可能暂时失效。
- 扩展重新加载后，SideMind 会在首次读取时自动补注入当前版本的内容脚本，无需手动刷新普通网页。
- 信息图、海报和扫描页中的文字属于图片像素，不是 DOM 正文；请点击截图按钮交给视觉模型理解。
- 工件只保存在 `chrome.storage.local`，最多保留最近 40 个并受 4MB 预算限制。为了避免模型生成代码发起隐藏请求，预览默认禁用脚本和外部网络资源。

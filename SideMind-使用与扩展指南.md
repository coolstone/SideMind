# SideMind 使用与扩展指南

适用版本：SideMind v0.5.16

SideMind 是一个运行在 Chrome 侧边栏里的个人 AI 助手。它可以读取当前网页、选区、本地文本文件和截图，把内容交给你配置的模型，并把回答写回网页输入框。

## 为什么做 SideMind

SideMind 最初不是为了做一个商业化的“Sider 替代品”，而是源于我作为 Sider 早期用户的一段真实使用经历。

在我的订阅经历中，Sider 最开始的年度订阅最高约为 999 元；后来 1399 元不再是最高档，而变成了第二档；如今最低档也已经到了 1699 元。产品持续升级和调整价格可以理解，但这样的变化让早期长期支持它的忠实用户越来越难以继续承担，也很难再用原来的方式支持下去。

我真正舍不得的不是某一个品牌，而是已经形成的工作方式：浏览网页时不离开当前页面，就能让 AI 阅读正文、处理选区、理解截图、调用提示词、继续追问，并把结果写回正在使用的工具。因此，我开始做一个主要服务于自己、成本和数据路径都可以掌控、还能随时修改的浏览器 AI 助手，这就是 SideMind 的由来。

SideMind 的功能来自长期使用中最常出现的实际需求：

- **浏览器侧边栏**：AI 应该在当前工作现场出现，而不是要求用户不断切换网页和复制粘贴。
- **读取网页与选区**：总结、解释、翻译和改写都应直接使用正在浏览的内容。
- **提示词管理**：高频工作流需要保存、排序、复用，并通过 `${lang}`、`${input}` 等变量接收当前上下文。
- **多模型配置**：用户可以使用自己的 DeepSeek、OpenAI、OpenRouter 或兼容接口，根据任务选择成本和能力合适的模型。
- **ChatGPT 网页交接**：在不读取 Cookie、不调用私有接口的前提下，也能利用已经登录的正常 ChatGPT 页面和订阅。
- **图片、截图和本地文件**：网页正文不总是完整的，图片、本地 Markdown 和文本附件也应该进入同一条工作流。
- **聊天空间与历史导出**：研究过程需要长期保存、搜索、改名和带走，而不是被锁在某个平台里。
- **写回网页**：回答不应停在聊天窗口里，还要能进入 Markdown 编辑器、CMS、表单和其他生产工具。
- **本地工件**：模型生成的 HTML、SVG、Markdown 和代码，应当可以立即预览、修改和保存。

SideMind 借鉴的是成熟产品已经验证过的浏览器交互方式，但代码、品牌、界面和功能实现都是围绕个人需求重新设计和独立完成的。它目前首先是一个个人工具：不追求一次覆盖所有商业产品能力，而是优先保证核心路径简单、可控、本地优先，并方便继续扩展。

SideMind 想解决的核心问题可以概括为一句话：**把模型能力留在用户自己的浏览器工作流里，让工具的成本、模型和未来功能重新由用户决定。**

## 一、使用压缩包安装到 Chrome

### 1. 解压到固定目录

Chrome 不能直接安装 ZIP 文件，必须先解压。

1. 找到 `SideMind-v0.5.16.zip`。
2. 解压到一个长期保留的目录，例如：

   ```text
   Documents/ChromeExtensions/SideMind
   ```

3. 打开解压后的目录，确认最外层可以看到 `manifest.json`、`sidepanel.html`、`background.js` 等文件。

不要每次升级都换一个新的安装目录。继续使用同一路径并覆盖文件，可以保持扩展身份以及本机保存的模型配置、聊天历史和提示词。

如果把新版解压到另一个目录后重新执行“加载已解压的扩展程序”，Chrome 可能把它识别为另一个扩展 ID。`chrome.storage.local` 按扩展 ID 隔离，此时原来的模型配置和聊天历史不会显示，但数据并不是被新版主动删除。历史可通过此前导出的 SideMind Markdown 文件导入；API Key 不会包含在导出文件中，需要重新填写。

### 2. 加载扩展

1. 在 Chrome 地址栏输入：

   ```text
   chrome://extensions/
   ```

2. 打开右上角的“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择刚才解压的 `SideMind` 文件夹，即包含 `manifest.json` 的那一层。
5. 安装成功后，在 Chrome 工具栏的扩展菜单中找到 SideMind，并点击图钉固定。
6. 点击 SideMind 图标，浏览器右侧会打开侧边栏。

### 3. 以后如何升级

1. 退出或关闭正在使用的 SideMind 侧边栏。
2. 用新版压缩包中的文件覆盖原安装目录。
3. 返回 `chrome://extensions/`。
4. 点击 SideMind 卡片上的“重新加载”。
5. 刷新已经打开的网页。

### 4. 允许读取本地文件（可选）

如果需要读取 `file:///...` 打开的 Markdown、TXT、CSV、JSON 或 HTML 文件：

1. 在 `chrome://extensions/` 找到 SideMind。
2. 点击“详情”。
3. 开启“允许访问文件网址”。
4. 刷新已经打开的本地文件页面。
5. 在 SideMind 当前网页卡片中点击刷新按钮。

SideMind 只读取浏览器当前打开的本地文件，不会扫描本机目录。

### 5. 建立本地数据备份

首次安装时 SideMind 会自动打开设置页，并提示建立或恢复本地备份。进入设置页的“本地数据与备份”后：

1. 点击“导出本地备份”，把 JSON 文件保存到长期保留的目录。
2. 备份包含模型配置、响应语言、启动偏好、提示词、聊天空间、聊天历史和本地工件。
3. API Key 默认不会写入备份。如果确实需要完整迁移，可以勾选“在备份中包含 API Key”；导出的 JSON 将以明文保存密钥，必须妥善保管。
4. 换目录、重装或扩展身份变化后，点击“恢复本地备份”并选择该 JSON 文件。

正常覆盖同一个安装目录并在 `chrome://extensions/` 点击“重新加载”时，Chrome 会保留 `chrome.storage.local`，无需每次恢复备份。本地备份主要用于误删扩展、加载了另一个目录、迁移电脑或浏览器配置损坏后的恢复。

## 二、配置 DeepSeek API Key

### 1. 获取 API Key

1. 登录 [DeepSeek 开放平台](https://platform.deepseek.com/)。
2. 在 API Keys 页面创建一个新的 Key。
3. 复制 Key，并确认账户有可用余额。

API Key 通常只会完整显示一次。不要把它写进 SideMind 源码，也不要放进需要分享的截图或聊天导出文件。

### 2. 在 SideMind 中填写

1. 打开 SideMind。
2. 点击侧边栏右上角的齿轮，进入设置。
3. 按下表填写：

| 配置项 | 推荐值 |
| --- | --- |
| 连接方式 | API Key · 回答直接返回 SideMind |
| 服务商 | DeepSeek |
| API 类型 | Chat Completions |
| Base URL | `https://api.deepseek.com` |
| 模型 | 下拉选择 `deepseek-v4-flash` |
| 最大输出 Token | 日常使用建议 `4000`—`16000` |
| API Key | 粘贴刚创建的 DeepSeek Key |

`deepseek-v4-flash` 速度和成本更适合日常网页问答；需要更强分析时可直接切换到内置的 `deepseek-v4-pro`。设置页采用“服务商 → 模型”两层结构：先配置 DeepSeek 的接口、Base URL 和 API Key，再管理其下模型。点击“添加模型”还可以继续保存其他模型或自定义模型 ID，同一服务商不需要重复填写 Key。

4. 点击“保存并测试连接”。
5. 页面显示“连接成功”后，返回侧边栏即可开始使用。

DeepSeek 官方当前 Base URL 仍为 `https://api.deepseek.com`，模型可使用 `deepseek-v4-flash` 或 `deepseek-v4-pro`。具体额度、价格和模型状态以 [DeepSeek API 文档](https://api-docs.deepseek.com/) 为准。

### 3. DeepSeek 图片限制

DeepSeek V4 官方 API 当前只支持文本输入，不能直接理解图片。SideMind 检测到 DeepSeek 和图片同时使用时，会保留图片并提示切换模型。

需要识图时可选择：

- 切换到支持视觉输入的 OpenAI、OpenRouter 或其他兼容模型；
- 切换为“网页”模式，把图片与提示词交接到已登录的 ChatGPT 页面；
- 先用其他视觉模型识别图片，再把识别文字交给 DeepSeek 继续分析。

## 三、开始第一次对话

1. 打开需要阅读的网页。
2. 点击 SideMind 图标。
3. 确认侧边栏顶部“当前网页”显示了正确标题。
4. 保持当前网页开关为开启状态。
5. 在输入框输入问题，例如：

   ```text
   总结当前页面，先给结论，再列出关键数据和待验证问题。
   ```

6. 按 Enter 或点击发送按钮。

`Shift + Enter` 用于输入换行。回答生成期间，右下角发送箭头会变为停止方块；点击即可强制中止当前模型请求。停止后问题仍保留在本地聊天记录中，可以修改后再次发送。

### 常用入口

鼠标停留在输入框底部的任一图标或选择器上，会显示功能名称和简要说明；使用键盘 Tab 聚焦控件时也会显示相同提示。

- **总结整页**：点击快捷提示词“总结”或输入自己的问题。
- **处理选区**：在网页选中文字，使用浮动工具条或右键菜单进行总结、解释、翻译和改写。
- **读取本地文档**：在 Chrome 打开本地 Markdown/TXT 等文件，并开启文件网址权限。
- **发送图片**：点击回形针选择图片，或在输入框按 `⌘V / Ctrl+V` 粘贴截图。
- **读取页面截图**：点击输入框工具栏的截图按钮，截取当前标签页的可见区域。首次升级到 v0.5.12 后需在扩展管理页重新加载扩展。
- **重新读取页面**：点击书本按钮或当前网页卡片的刷新按钮。
- **深度分析**：点击“思考”，在关闭、高、最大之间切换。
- **停止生成**：API 请求进行中，点击输入框右下角的停止方块，立即取消当前网络请求。
- **开始新聊天**：点击输入框左下角的 `+`。
- **查看历史**：点击 `+` 旁边的历史图标。

## 四、主要功能介绍

### 1. 浏览器上下文

- 自动读取网页标题、描述、标题结构和正文。
- 实时读取网页选区。
- 支持当前标签页截图和图片理解。
- 支持浏览器打开的本地 Markdown、TXT、CSV、JSON、HTML 文件。
- 可关闭“当前网页”开关，进行不带网页内容的普通聊天。

页面截图使用 Manifest V3 的 `<all_urls>` 持续页面访问权限，避免侧边栏跨标签页或页面跳转后 `activeTab` 临时授权失效。安装或升级后，到 `chrome://extensions/` 对 SideMind 点击一次“重新加载”，再刷新目标网页即可；不需要开启 macOS 的屏幕录制权限。Chrome 设置页、扩展商店等受保护页面仍不能截图。本地 `file://` 文件还需要在扩展详情中单独开启“允许访问文件网址”。截图只包含当前屏幕可见区域，并且只有支持视觉输入的模型才能理解图片内容。

### 2. 多模型和两种连接模式

- 设置页先配置服务商的 API 类型、Base URL 和 API Key，再管理该服务商下的模型。
- 同一服务商的模型集中显示并共享连接配置；默认提供 DeepSeek V4 Flash 与 V4 Pro 两个模型档案。
- 支持 OpenAI Responses API。
- 支持 DeepSeek、OpenRouter 和自定义 OpenAI 兼容 Chat Completions 接口。
- 支持 Ollama 本地服务；Base URL 填写到 `http://主机:11434` 即可，SideMind 会自动请求 `/v1/chat/completions`，API Key 可留空。
- 侧栏底部可直接切换模型，弹出的列表按服务商分组。
- ChatGPT 网页交接模式不读取 Cookie、不调用私有接口，提示词填入后由用户确认发送。

### 3. 调整侧栏字体

打开“设置 → 侧栏字体大小”，可选择紧凑、舒适、大或特大。默认使用“舒适”，比旧版的聊天正文、输入框和小型操作文字更大；保存后已打开的侧栏会立即应用，不需要重新安装扩展。

### 4. 提示词管理

- 内置总结、解释、翻译、写作、词汇学习和三阶段行业分析等模板。
- 支持创建、修改、删除、搜索和分类。
- 支持拖动或按钮调整显示顺序。
- 输入框上方显示常用提示词，“…”打开完整列表。
- `${lang}` 自动使用设置中的响应语言。
- `${input}` 优先使用网页选区或当前输入框文字。
- 自定义 `${变量}` 会在运行提示词时生成输入项。

### 4. 附件与图片

- 支持 PNG、JPEG、WebP 图片。
- 支持 TXT、Markdown、CSV、JSON、HTML 文本附件。
- 支持在输入框直接粘贴剪贴板图片并显示缩略图。
- 单文件最大 3MB，一次最多 6 个附件，总量最大 5MB。
- 模型必须具备相应的视觉能力，才能理解图片内容。

### 5. 聊天空间和历史

- 不同项目可以建立不同聊天空间。
- 聊天历史在本机浏览器中保存。
- 支持修改聊天标题和全文搜索。
- 可配置启动时恢复上次会话或始终新建会话。
- 恢复历史后会定位并高亮当前会话。
- “导入记录”可选择 SideMind 导出的 Markdown 文件，把其中会话去重合并到当前空间。
- “导出当前空间”可生成包含标题、时间、关联网页和完整问答的 Markdown 文件。
- 设置页可导出或恢复完整本地 JSON 备份；它比聊天 Markdown 多包含模型配置、提示词、空间、偏好和本地工件。

### 6. 回答下方工具栏

- 复制回答。
- 下载为本地 Markdown。
- 写入网页输入框。
- 重新生成最新回答。
- 引用回答继续提问。
- 调用系统分享。
- 语音朗读。
- 确认后删除回答。
- 代码块可单独复制。

### 7. 本地工件

- 自动识别回答中的 HTML、SVG、Markdown、JSON、CSS、JavaScript/TypeScript、XML、Vue 等代码块。
- 生成独立工件卡片。
- 支持代码、预览和分栏模式。
- 支持本地修改、自动保存、复制和下载。
- HTML/SVG/Markdown 预览运行在隔离沙箱中。

### 8. 自愿捐赠

- 点击 SideMind 顶部的爱心按钮打开“支持 SideMind”。
- 可在微信支付与支付宝之间切换并扫描对应二维码。
- 捐赠完全自愿，不会解锁、限制或改变任何产品功能。
- 二维码图片随扩展保存在本地，SideMind 不会上传二维码或收集支付信息。

## 五、与在线 Markdown 工具联动

截图中的典型流程是：在线 Markdown 工具负责编辑和预览，SideMind 负责阅读、分析、改写和生成内容。

### 1. 让 SideMind 读取工具页面

1. 打开在线 Markdown 编辑器或本地部署的文档工具，例如 `http://183.6.65.3:8282/`。
2. 打开 SideMind，确认当前网页标题正确。
3. 点击“阅读此页”重新获取页面中可见的正文、标题和预览内容。
4. 提问，例如：

   ```text
   检查当前 Markdown 文档的结构，指出重复内容、缺失章节和表达不清的地方。
   ```

### 2. 把回答写回编辑器

1. 先在网页中点击需要写入的 Markdown 编辑框，让它获得焦点。
2. 回到 SideMind 回答下方，点击“写入网页输入框”按钮（方框加号图标）。
3. SideMind 会把当前回答插入刚才获得焦点的网页输入框。
4. 在 Markdown 工具中继续编辑、保存或导出。

SideMind 会记住最近一次获得焦点的普通输入框、文本域或 `contenteditable` 编辑器。如果编辑器位于跨域 iframe、Canvas 或自定义封闭组件中，浏览器安全策略可能阻止直接写入，此时使用“复制回答”再手动粘贴。

### 3. 推荐工作循环

```text
Markdown 工具打开文档
        ↓
SideMind 读取当前页面
        ↓
总结 / 检查 / 改写 / 生成新章节
        ↓
写入网页输入框
        ↓
Markdown 工具预览与导出
```

这个方式也适用于 CMS 后台、博客编辑器、客服系统、表单、知识库和代码编辑页面。

## 六、方便自己继续扩展

SideMind 是原生 Manifest V3 扩展，没有 npm 依赖，也没有构建步骤。修改源文件后，在 `chrome://extensions/` 点击“重新加载”即可。

### 1. 主要文件

| 文件 | 用途 |
| --- | --- |
| `manifest.json` | 扩展权限、版本、内容脚本和侧边栏入口 |
| `background.js` | 模型请求、截图、右键菜单、侧边栏和 ChatGPT 交接 |
| `content.js` | 网页正文提取、选区工具条、写回网页、ChatGPT 页面桥接 |
| `content.css` | 注入网页划词工具条和 ChatGPT 网页交接样式 |
| `sidepanel.html/css/js` | SideMind 主侧栏界面和交互 |
| `options.html/css/js` | 模型、API Key、响应语言和启动行为设置 |
| `prompts.html/css/js` | 提示词管理界面 |
| `prompt-library.js` | 内置提示词库 |
| `template-variables.js` | `${lang}`、`${input}` 和自定义变量替换 |
| `artifact.html/css/js` | 本地工件编辑与预览 |
| `tests/regression.mjs` | 无依赖回归检查 |

### 2. 不改代码也能扩展

- 在提示词管理中创建自己的工作流模板。
- 用 `${input}` 接收网页选区或输入框内容。
- 用 `${industry}`、`${tone}` 等变量创建运行时表单。
- 建立多个聊天空间区分不同项目。
- 新建多个模型配置，分别用于日常、深度分析和视觉任务。

### 3. 增加一个新的快捷能力

1. 在 `sidepanel.js` 的 `ACTIONS` 中增加动作名称、显示文本和任务提示词。
2. 在需要显示按钮的位置增加对应的 `data-action`。
3. 如需读取选区，复用 `state.context.selection`。
4. 运行回归检查并重新加载扩展。

如果只是增加提示词，优先使用提示词管理，不必修改 `ACTIONS`。

### 4. 增加模型服务商

1. 在 `options.html` 增加服务商选项。
2. 在 `options.js` 的 `PROVIDER_PRESETS` 中增加默认 API 类型、Base URL、模型和 Token 提示。
3. 在 `background.js` 中确认该服务兼容 Responses 或 Chat Completions 请求体。
4. 如果服务商的思考或图片参数不同，在 `buildResponsesBody` 或 `buildChatBody` 中做精确适配。
5. 使用“保存并测试连接”验证最小文本请求。

不要仅因为接口名称写着“OpenAI 兼容”就假定它支持图片、思考参数或相同的最大 Token。

### 5. 为特定网页增加写入适配

通用写入逻辑位于 `content.js` 的 `insertIntoPage`。如果某个在线工具使用特殊编辑器，可以：

1. 根据 `location.hostname` 识别站点。
2. 查询该站点真正接收输入的 `textarea` 或 `contenteditable` 元素。
3. 使用原生属性 setter 写入内容。
4. 触发 `input` 和 `change` 事件，让 React/Vue 等框架同步状态。
5. 保留通用逻辑作为回退。

跨域 iframe 无法由父页面内容脚本直接访问，需要在 iframe 对应域名中也注入内容脚本，或改用复制粘贴流程。

### 6. 增加工件类型

在 `sidepanel.js` 的 `buildArtifact` 类型表中增加扩展名、MIME 类型、显示名称以及是否支持预览。需要执行代码或访问网络的预览必须继续使用隔离策略，不能直接在 SideMind 扩展页面运行模型生成的脚本。

### 7. 修改后验证

在项目目录运行：

```bash
node --check background.js
node --check content.js
node --check sidepanel.js
node --check options.js
node --check prompts.js
node tests/regression.mjs
```

然后：

1. 在 `chrome://extensions/` 重新加载扩展。
2. 刷新测试网页。
3. 右键侧边栏选择“检查”，查看侧栏控制台。
4. 在扩展卡片点击 Service Worker，查看后台错误。

## 七、常见问题

### Chrome 无法选择 ZIP

这是正常现象。请先解压，再选择包含 `manifest.json` 的文件夹。

### 升级后功能没有变化

确认新版文件已覆盖原目录，然后在 `chrome://extensions/` 点击“重新加载”，并刷新目标网页。

### 本地 Markdown 显示“无法读取”

在扩展详情中开启“允许访问文件网址”，然后刷新本地文件页。

### DeepSeek 返回 Authentication Fails

检查 API Key 是否复制完整、是否属于 DeepSeek 开放平台，以及账户是否有余额。ChatGPT 订阅不能替代 DeepSeek 或 OpenAI API 额度。

### DeepSeek 无法分析图片

DeepSeek V4 官方 API 当前为纯文本输入。请切换到支持视觉的模型或 ChatGPT 网页模式。

### Ollama 测试连接返回 HTTP 403

这通常不是模型或 API Key 错误，而是 Ollama 拒绝了浏览器扩展的 `chrome-extension://` 请求来源。SideMind 已经会把裸地址（例如 `http://192.168.110.4:11434`）自动转换为正确的 `/v1/chat/completions`，仍需在 Ollama 服务端放行来源：

1. 在 `chrome://extensions/` 找到 SideMind，复制扩展 ID。
2. 在运行 Ollama 的电脑上设置 `OLLAMA_ORIGINS`，然后完全重启 Ollama。
3. 个人开发环境可使用 `OLLAMA_ORIGINS=chrome-extension://*`；希望限制更严时，只填写 `chrome-extension://你的扩展ID`。
4. 如果 SideMind 与 Ollama 不在同一台电脑，还需让 Ollama 监听局域网地址，例如配置 `OLLAMA_HOST=0.0.0.0:11434`，并确认系统防火墙只允许可信局域网访问。
5. 在 SideMind 设置中选择“服务商 → Ollama（本地模型）”，Base URL 填 `http://192.168.110.4:11434`，API Key 留空，模型 ID 填写 `gemma4:31b`，再点击“保存并测试连接”。

macOS Ollama 应用的环境变量可以通过 `launchctl setenv OLLAMA_ORIGINS "chrome-extension://*"` 设置，之后需要从菜单栏完全退出并重新打开 Ollama。使用 systemd、Docker 或其他启动方式时，应把同名环境变量写入对应服务配置并重启容器或服务。不要把未鉴权的 Ollama 端口直接暴露到公网。

### “写入网页输入框”失败

先点击目标网页编辑框再执行写入。若页面使用跨域 iframe、Canvas 编辑器或受保护的浏览器页面，请使用复制回答并手动粘贴。

### 网页正文只有标题

页面可能主要由图片、Canvas、iframe 或延迟加载内容组成。可以先点击“阅读此页”，仍无正文时附加当前页面截图交给视觉模型。
